> **Mini-LLM de 5.26M parámetros, escrito a mano en JAX puro, que aprende
> _data-to-text_: tomá los stats de un partido de fútbol y produce la
> crónica en español en uno de 8 estilos regionales de comentarista.**

Inputs a structured `<STATS>` block + a style, generates a Spanish crónica.

## What this project is — and isn't

This is a **portfolio piece that demonstrates the craft of implementing a
Transformer from scratch in pure JAX**. The data-to-text task is a real
application; the technical core (attention, RoPE, RMSNorm, SwiGLU, training
loop, sampling) is hand-written precisely because the goal is to show
understanding at the math level, not to call HuggingFace APIs.

**It is not** a state-of-the-art language model. A 5M-param model fine-tuned
from Qwen-3B with LoRA would produce far better crónicas. We chose
from-scratch JAX because that's the signal that matters for ML/research
roles where deep understanding is valued.

### What "from scratch" means here

What we **did** build from scratch:
- **The architecture**: hand-rolled in pure JAX (no Flax, no Equinox). Every
  line of attention, RoPE, RMSNorm, SwiGLU, the training step, sampling is
  written out explicitly in `src/cronica/`.
- **The weights**: initialized with `jax.random.truncated_normal(std=0.02)`.
  **No pretrained checkpoint was loaded.** This is not fine-tuning of Llama
  or GPT-2; it is random init → 2,000 steps → checkpoint.
- **The tokenizer**: byte-level BPE trained on our corpus, not borrowed.
- **The training loop**: our own jit'd `value_and_grad` + Optax AdamW + loss
  masking. Not a wrapper around HF Trainer.

What we **did not** do, and we say so honestly:
- The training **targets** (the crónicas) are **synthetic**, generated by
  OpenAI `gpt-4o-mini` against the structured `<STATS>` inputs. So this is
  a form of **knowledge distillation** — the small JAX model is learning
  to imitate the larger teacher on a structured task. The same pattern
  is used by [DistilBERT](https://arxiv.org/abs/1910.01108),
  [Alpaca](https://crfm.stanford.edu/2023/03/13/alpaca.html),
  [Phi](https://arxiv.org/abs/2306.11644). It is legitimate; the framing
  matters.
- We are **not** claiming this competes with billion-parameter models. With
  perplexity 16.4 on Spanish text, this is a small student model.

## End-to-end pipeline

```
Kaggle: davidcariboo/player-scores (CC0)
   └─ 82,073 matches with at least 1 goal
       └─ build_prompts.py renders <STATS> blocks in Spanish
           └─ synth_cronicas.py / synth_batch.py — OpenAI gpt-4o-mini
               using 8 commentator-persona prompts (private) → 8
               public-facing style labels
                   └─ data/synthetic/pairs.jsonl (5,000 pairs, ~1.4M tokens)
                       └─ tokenizer.py — byte-level BPE, vocab=8k,
                          16 special tokens (4 sentinel + 4 delimiters
                          + 8 style tokens)
                           └─ train.py — pure-JAX Transformer, 5.26M
                              params, AdamW + cosine + grad-clip,
                              **loss masking** so the model is only
                              penalized on the cronica span
                               └─ checkpoint @ step 2000
                                   └─ sample.py — top-k/top-p with
                                      <STATS> prompt conditioning
                                       └─ HF Hub model + dataset push
                                           └─ HF Space (Gradio demo)
```

## Architecture

| component        | value                                                |
|------------------|------------------------------------------------------|
| Type             | Decoder-only Transformer                             |
| Params           | **5.26M**                                            |
| vocab            | 8000 (byte-level BPE) + 16 special tokens            |
| d_model          | 256                                                  |
| n_layers         | 4                                                    |
| n_heads          | 4 (d_head=64)                                        |
| d_ff             | 704                                                  |
| max_seq_len      | 768                                                  |
| Positional enc.  | **RoPE**                                             |
| Norm             | **RMSNorm** (pre-norm)                               |
| MLP              | **SwiGLU**                                           |
| Embeddings       | Tied input/output                                    |
| Optimizer        | Optax AdamW, lr_peak=3e-4, cosine, warmup=100, wd=0.1 |
| Grad clip        | global_norm=1.0                                      |
| Loss             | Cross-entropy **masked** to the `<cronica>` span     |

**Implementation**: `src/cronica/model.py` is **pure JAX** — params are
PyTree dicts of `jnp.ndarray` and the forward pass is a plain functional
`apply(params, tokens, cfg) -> logits`. No Flax, no Equinox, no helper
libraries. Everything from RoPE rotation to the SwiGLU gating is written
out explicitly.

## Training run (Apple M4 CPU, no GPU/TPU)

- **5,000 pairs** drawn from
  [`DanielRegaladoCardoso/cronicas-d2t`](https://huggingface.co/datasets/DanielRegaladoCardoso/cronicas-d2t)
- **2,000 gradient steps**, batch_size=8, seq_len=768
- **~28 minutes** total wall-clock on M4 CPU (10 cores)
- **~6,940 tokens/sec** sustained
- 4 checkpoints saved (every 500 steps)

### Loss curve

| step | loss avg |  step | loss avg |
|------|----------|-------|----------|
|  25  | 8.74     |  500  | ~3.50    |
|  100 | 6.37     | 1000  | ~3.00    |
|  150 | 5.34     | 1500  | 2.95     |
|  300 | ~4.00    | **2000** | **2.80** |

Random baseline = `ln(8000) ≈ 8.99`. Final perplexity ≈ **16.4**.
Clean monotonic descent, no divergence, no signature of catastrophic
overfit despite the small data-per-param ratio.

## Eight commentator styles

Synthetic data was generated with prompt variants that reference real
broadcasters internally, then mapped to descriptive public labels to
avoid likeness issues. See
[`docs/decisions/002-commentator-personas.md`](docs/decisions/002-commentator-personas.md).

| style                        | flavor                                  |
|-----------------------------|------------------------------------------|
| `rioplatense_apasionado`    | Argentina, emotive, "GOOOL"              |
| `rioplatense_tecnico`       | Argentina, analytical                    |
| `rioplatense_literario`     | Uruguay, evocative prose ("barrilete")   |
| `mexicano_irreverente`      | Mexico, sarcastic                        |
| `mexicano_clasico`          | Mexico, formal                           |
| `centroamericano_espn`      | El Salvador / ESPN Latam, polished       |
| `espanol_radiofonico`       | Spain, COPE radio style                  |
| `comentario_tecnico`        | tactical analysis                        |

## Held-out evaluation (metrics that don't lie)

12 crónicas generated against 4 matches the model had never seen, across
3 styles. Full output in [`results/samples.md`](results/samples.md), raw
metrics in [`results/metrics.json`](results/metrics.json).

### Style fidelity (the strong story)

A crónica counts as "style activated" if it contains at least one
distinctive marker for that persona (`gooool` for apasionado, `barrilete`
or `magia` for literario, `transiciones` or `sistema` for técnico).

| style                       | activation | n |
|----------------------------|------------|---|
| `rioplatense_apasionado`   | **100.0%** | 4 |
| `rioplatense_literario`    | **100.0%** | 4 |
| `comentario_tecnico`       | **50.0%**  | 4 |

The two distinctive styles are activated on every held-out sample. The
model genuinely learned the difference between an Andrés Cantor-flavored
crónica and a Víctor Hugo Morales-flavored one.

### Grounding (the honest story)

A crónica counts as "grounded" on each input field if the field's content
appears in the generated text.

| metric                          | held-out | training data |
|--------------------------------|----------|---------------|
| Home team referenced            | **16.7%** | 98.0%        |
| Away team referenced            | **25.0%** | 97.5%        |
| Final score literal             | **0.0%**  | 70.7%        |
| Scorer recall (last names)      | **0.0%** (0/39) | 95.7% |

The student model learned **style** but mostly failed at **grounding**.
At 5M params and ~1.4M training tokens, the model captured "what an
apasionado crónica sounds like" but not "what these specific players
did in this specific match". This is the expected behavior of a small
distilled model on a structured task — it picks up surface stylistics
faster than fine-grained input copying.

### Example output

> **Style**: `rioplatense_literario` | **Stats**: Match in Bundesliga
>

## Tests

19 hard-assert integration tests in `tests/test_e2e.py`. Coverage:

- Dataset shape: count, fields, style balance, Spanish content, length
- Tokenizer: vocab size, all 16 special tokens single-id, zero `<unk>` on
  500-doc sample, encode-decode roundtrip preserves word count
- Model: checkpoint loads, params 4M–6.5M, config consistency, forward
  produces finite logits on zero and random input
- Sampling: produces text, no leaked special tokens, rejects unknown style
- Pipeline wiring: `<cronica>` boundary in loss mask, prompt tokens
  masked off, pad positions masked off

```bash
PYTHONPATH=src pytest -v tests/
# ====== 19 passed in 30.42s =====
```

## Honest limitations

- **Small model on small data.** 5M params on 1.4M tokens is ~1000×
  over-parameterized vs. Chinchilla-optimal. Expect grammatical sentences
  but limited paragraph-level coherence and hallucinated team/scorer names.
- **Coverage bias.** Training set is weighted toward big European
  leagues + CONMEBOL. Some leagues (Liga MX in particular) are not in
  the Transfermarkt slugs we mapped to Spanish.
- **Style overlap.** Of the 8 styles, three carry strong distinctive
  vocab (`rioplatense_apasionado` → "gooool", `rioplatense_literario`
  → "barrilete", `comentario_tecnico` → "transiciones"). The other five
  share more neutral journalistic vocabulary.
- **Hallucinated context.** The teacher model (`gpt-4o-mini`) sometimes
  added unfamiliar manager names, derby names, or stadium nicknames not
  present in `<STATS>`. The small JAX model reproduces some of this
  hallucination tendency.

These are documented honestly here, in the model card, and in the dataset
card — engineering judgment is part of the portfolio piece.

## Repo layout

```
src/cronica/
  model.py           # pure-JAX Transformer
  tokenizer.py       # byte-level BPE wrapper
  train.py           # jit'd training loop with loss masking
  sample.py          # greedy / top-k / top-p sampling
  data.py            # pairs loader, mask construction

scripts/
  load_kaggle.py     # download & filter rich matches
  build_prompts.py   # render <STATS> blocks in Spanish
  synth_cronicas.py  # OpenAI realtime API (8 personas)
  synth_batch.py     # OpenAI Batch API (50% cheaper)
  clean.py           # quality + dedup filters (legacy)
  diagnose_data.py   # data quality + tokenization diagnostic
  analyze_corpus.py  # pre-pivot corpus analysis
  push_dataset.py    # validates then pushes pairs to HF Hub
  push_model.py      # validates then pushes ckpt to HF Hub
  push_space.py      # validates then pushes Gradio Space
  eval_heldout.py    # generate held-out crónicas
  compute_metrics.py # grounding + style fidelity over samples.md

space/             # Gradio app for HF Space
docs/decisions/    # ADRs (architecture decisions)
tests/             # 19 e2e tests
results/           # held-out samples + metrics.json
```

## How to reproduce

```bash
# Clone
git clone https://github.com/DanielRegaladoUMiami/cronica-jax
cd cronica-jax
pip install jax jaxlib optax tokenizers huggingface-hub gradio pandas \
            pyarrow kagglehub httpx beautifulsoup4 pytest

# 1. Source data (~3 min, needs kaggle credentials in ~/.kaggle/kaggle.json)
python -m scripts.load_kaggle --out data/kaggle

# 2. Render prompts in Spanish (~30 s, 5k from filtered allow-list)
python -m scripts.build_prompts --max 5000

# 3. Generate synthetic crónicas with OpenAI (cost ~$1, ~15 min)
export OPENAI_API_KEY=...           # in ~/.zshrc, NOT in the shell
python -m scripts.synth_batch submit --in data/prompts/prompts.jsonl --max 5000
python -m scripts.synth_batch status --watch
python -m scripts.synth_batch fetch --batch-id <id> --out data/synthetic/pairs.jsonl

# 4. Train BPE
PYTHONPATH=src python -m cronica.tokenizer train \
  --input data/synthetic/pairs.jsonl --out tokenizer.json --vocab-size 8000

# 5. Train (~28 min on Apple M4 CPU)
PYTHONPATH=src python -m cronica.train \
  --pairs data/synthetic/pairs.jsonl --tokenizer tokenizer.json \
  --out-dir checkpoints/run01 --steps 2000 --batch-size 8 --seq-len 768

# 6. Sample
PYTHONPATH=src python -m cronica.sample \
  --ckpt checkpoints/run01/ckpt_002000.pkl --tokenizer tokenizer.json \
  --stats examples/match.txt --style rioplatense_apasionado

# 7. Eval + metrics
PYTHONPATH=src python -m scripts.eval_heldout \
  --ckpt checkpoints/run01/ckpt_002000.pkl --tokenizer tokenizer.json \
  --prompts data/prompts/prompts_heldout.jsonl --n 8
PYTHONPATH=src python -m scripts.compute_metrics

# 8. Tests
PYTHONPATH=src pytest -v tests/

# 9. Push artifacts (require HF auth)
PYTHONPATH=src python -m scripts.push_dataset --pairs data/synthetic/pairs.jsonl
PYTHONPATH=src python -m scripts.push_model   --ckpt checkpoints/run01/ckpt_002000.pkl
PYTHONPATH=src python -m scripts.push_space
```
